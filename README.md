# 🎮 Project: Item Simulator
---

## 🔍 게임 소개
**아이템 시뮬레이터**는 게임 속 캐릭터와 아이템을 관리하는 기능을 구현한 프로젝트입니다. 아래와 같은 주요 기능들을 제공합니다:

### 🛠️ 구현된 기능
1. **계정 생성**  
   - URL: `/api/account`

2. **계정 삭제**  
   - URL: `/api/account/:userID`

3. **캐릭터 생성**  
   - URL: `/api/character`

4. **캐릭터 삭제**  
   - URL: `/api/character/:name`

5. **캐릭터 상세 조회**  
   - URL: `/api/character/:name`

6. **아이템 생성**  
   - URL: `/api/item`

7. **아이템 삭제**  
   - URL: `/api/item/:code`

8. **아이템 상세 조회**  
   - URL: `/api/item/:code`

9. **아이템 수정**  
   - URL: `/api/item/:code`

---

### 🚨 주의사항
- **JWT 인증 과정은 미구현 상태**입니다.  
  추후 보안 강화를 위해 인증 절차를 추가할 예정입니다.

---

## 🔒 1. 암호화 방식
**Q. 비밀번호를 DB에 저장할 때 Hash는 단방향 암호화와 양방향 암호화 중 어느 방식에 해당하나요?**  
A. Hash는 **단방향 암호화**에 해당합니다.

**Q. 비밀번호를 Hash 값으로 저장하면 어떤 장점이 있나요?**  
A. 비밀번호가 유출되어도 복호화가 불가능해 보안성이 높습니다.

---

## 🔑 2. 인증 방식
**Q. JWT(Json Web Token) Access Token이 노출되면 어떤 문제가 생길 수 있나요?**  
A. 공격자가 노출된 토큰으로 사용자로 가장해 서비스를 악용할 수 있습니다.

**Q. 이 문제를 보완하는 방법은 무엇인가요?**  
A. 토큰의 유효 기간을 짧게 설정하는 방법이 있을 것 같습니다.

---

## 🛂 3. 인증과 인가
**Q. 인증과 인가의 차이는 무엇인가요?**  
- **인증**: 사용자가 누구인지 확인하는 것.  
- **인가**: 사용자에게 어떠한 작업에 대해 권한을 부여하는 것.

**Q. 인증이 필요한 API와 필요 없는 API의 차이는 무엇인가요?**  
A. 인증이 필요한 API는 보안이 중요하고, 인증이 필요 없는 API는 공개된 정보만 제공한다고 생각합니다.

**Q. 아이템 생성/수정 API에 인증이 왜 필요한가요?**  
A. 인증되지 않은 사용자가 악의를 품고 데이터를 조작하거나 훼손하면 가치가 있는 데이터까지 위협을 받을 수 있기 때문입니다.

---

## 📡 4. Http Status Code
**Q. 사용한 Http Status Code와 그 의미는 무엇인가요?**  
- **200 OK**: 요청이 성공했을 때.  
- **400 Bad Request**: 요청이 잘못되었을 때.  
- **500 Internal Server Error**: 서버에 문제가 발생했을 때.

---

## 💰 5. 게임 경제
**Q. 캐릭터 테이블에 `money` 컬럼만 추가했을 때 문제점은 무엇인가요?**  
A. 게임 머니 사용 기록이 없어 불법 행위의 추적이 어렵습니다.

**Q. 이를 해결하기 위한 방법은 무엇인가요?**  
A. 게임 머니 사용 기록을 관리하는 테이블을 추가 생성 관리합니다.

**Q. 아이템 가격을 클라이언트에서 입력하게 하면 어떤 문제가 생기나요?**  
A. 모든 아이템은 가치의 신뢰를 잃습니다. 이후 새로운 아이템이 나온다고 해도 신뢰가 회복되지 않는다면 게임 경제가 침체되어 재미없는 게임이 되어버립니다. 재미가 없는 게임은 게임이 아니라고 생각합니다.
